# quark-twitter-pipeline v2.2

## 用途

这个 skill 用于处理夸克网盘分享链接，完成以下固定流程：

1. 读取原始分享内容
2. 转存到当前账号网盘（默认 `pull` 文件夹）
3. 为转存后的文件创建新的分享链接（默认无提取码）
4. 生成多个专业 Twitter 文案版本（**自动带标签 + 配图建议**）

## 入口脚本

脚本路径：`scripts/quark_twitter_pipeline.py`

主命令：
```bash
python scripts/quark_twitter_pipeline.py run "<夸克分享链接>"
```

查看支持的文案风格：
```bash
python scripts/quark_twitter_pipeline.py styles
```

## 前置条件

1. 已安装 Python 3.7 或更高版本
2. 已安装依赖 httpx
3. 本机存在夸克 cookie 文件，默认路径为 `~/.quark_cookie.txt`

## 账号定位（资源猎人）

**核心定位：资源猎人，不是单纯搬运链接的人。**

### 主张
每天在互联网里找工具、资源和方法，但不做单纯搬运工。

### 内容方向（5大板块）
1. **好用工具**：AI、效率、下载、整理、办公
2. **网盘资源**：保存、转存、备份、资料管理
3. **避坑经验**：哪些会员值得买，哪些资源别乱囤
4. **数字生活**：普通人怎么管理文件、资料和信息
5. **日常碎念**：关注趋势，发表不成熟的看法

### 选品标准
- 不发看起来很爽、实际没用的垃圾资源
- 不鼓励无脑收藏
- 只分享真正能节省时间、解决问题、提升效率的东西

### Slogan
**Less noise, more value.**

## Twitter 发布规范（重要）

> 以下规范基于实际运营数据总结，每次生成都应遵守。

### 必须配图
- 发推**必须带图片**，纯文字+链接点击率极低
- 图片是吸引点击的关键，没有图片的推文基本没人看
- 配图建议：资源封面截图、成果展示图、工具界面截图
- 如果是视频/课程类资源，用课程封面或目录截图效果最好

### 固定开头格式
一句话说明白这是什么：
- 「今天分享一套XXX（视频+资料）」
- 「刚找到一套XXX，说说感受」
- 「压箱底的XXX，今天拿出来」
- 「你们催更的XXX终于来了」

### 文案结构（4段式）
```
开头（1句话，说清楚是什么）
↓
个人见解/使用体验（核心差异化内容）
↓
链接
↓
互动引导 + 标签
```

### 标签要求
- **每条推文必须带 2-3 个相关标签**
- 不要超过 3 个，多了像刷屏
- 标签要和资源强相关，不要蹭不相关热点
- 常用标签池：
  - #资源分享 #效率工具 #AI工具 #自学 #副业 #干货 #求职 #职场 #网盘 #数字生活

### 发布时间（北京时间）
- **优先时段**：9:00-11:00 或 20:00-22:00
- 避开凌晨和下午低谷期
- 周二到周四互动率通常较高

## 文案风格

| 风格 | 说明 | 推荐场景 |
|------|------|---------|
| auto | 自动生成4种风格版本（**默认**） | 不确定用哪种时 |
| urgent | 紧迫稀缺风格 - 制造紧迫感，限时稀缺 | 限时资源、热门资源 |
| value | 价值强调风格 - 突出资源价值和实用性 | 实用工具、教程类 |
| hype | 热度爆发风格 - 情绪饱满，引发转发 | 爆款资源、话题性强的 |
| professional | 专业简洁风格 - 简洁专业，适合行业交流 | 行业资料、专业课程 |
| story | 故事型风格 - 用故事引入，更有代入感 | 有故事线的内容 |
| ip | **个人IP风格** - 资源 + 个人见解/使用体验 | **日常发布首选** |

## 输入参数

### 必填参数

| 参数 | 说明 |
|------|------|
| link | 夸克分享链接，必须包含 /s/xxx 形式的分享 ID |

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|-------|
| --title, -t | 资源标题；未提供时自动推断 | 自动推断 |
| --desc, -d | 资源描述/价值亮点 | 空 |
| --opinion, -o | **个人见解/使用体验**（生成个人IP风格文案） | 空 |
| --style, -s | 文案风格 | auto |
| --tags | **自定义标签**（逗号分隔，如 "资源分享,AI工具"） | 自动生成相关标签 |
| --cookie | 直接传入 cookie 字符串 | - |
| --save-to | 转存目标文件夹 ID | 0（根目录） |
| --to-folder | 按文件夹名称查找目标目录（**默认：pull**） | pull |
| --cookie-file | cookie 文件路径 | ~/.quark_cookie.txt |

## 推荐调用方式

```bash
# 默认生成4种风格版本（含标签+配图建议）
python scripts/quark_twitter_pipeline.py run "https://pan.quark.cn/s/xxx"

# 个人IP风格 + 个人见解（推荐日常使用）
python scripts/quark_twitter_pipeline.py run "https://pan.quark.cn/s/xxx" \
    --opinion "用了三天，说说真实感受：✅ 文笔扎实 ✅ 视角新鲜 ✅ 熬夜看完的" \
    --style ip \
    --tags "资源分享,读书"

# 带描述 + 自定义标签
python scripts/quark_twitter_pipeline.py run "https://pan.quark.cn/s/xxx" \
    --desc "包含完整工具包和模板，从0到1" \
    --tags "AI工具,效率提升" \
    --style value
```

## 返回结果

正常情况下，返回以下信息：
1. 新的夸克分享链接（无提取码）
2. 4个 Twitter 文案版本（或指定风格的1个版本）
3. 每个文案的长度统计
4. **自动生成的相关标签建议**
5. **配图建议**

## 文案偏好（人设约束）

- 口语化、轻松、配合 emoji
- 不要官腔，不要装
- 可以有个人偏好和判断（"这个真的好用" vs "这个一般"）
- 金额保留2位小数
- 能短就短，但该展开的地方要展开

## 输出约束

- 文案基于脚本实际生成结果返回
- 每次生成内容随机组合，避免机械化重复
- 全部文案控制在280字以内
- **输出必须包含标签**
- **输出必须附带配图建议**
